概述

api manager 是用来做api管理，并进行请求代理转发的。

每一个请求会同时复制到所有的后端，然后选取其中一个后端的响应返回给client。

用途演示

多人多模块开发

1.一种情况：2个模块分别2个人开发，2个环境都能收到数据。

2.另一种情况：合作有交叉，下游一个人就收不到数据了。

3.一个理想的情况：合作有交叉时，下游每个人都能收到数据。

高级用法

动态指定Response后端：

若同时有多个后端（如 a,b,c 共3个），可以在http的 GET/POST、Header中设置参数api_pref=a,b来指定返回Response的后端。

设置cookie格式为:api_pref_{apiName}=a,b。

参数优先级：1-GET/POST > 2-Cookie > 3-Header > 4-IP。
